# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Personal dotfiles for an Arch Linux (CachyOS) + Hyprland/Wayland desktop, with partial macOS support. There is no build/test/lint pipeline — "running" this repo means deploying symlinks and installing packages onto a live machine, so most changes are validated by hand on the target system.

## Layout & how config is deployed

Three top-level config trees, each mirroring `$HOME`:

- `common/` — cross-platform (shells, terminals, git, editors, CLI tools).
- `linux/` — Hyprland desktop, Wayland tools, systemd user units, `~/.local/bin` scripts.
- `macos/` — aerospace, ghostty.

Deployment uses **GNU stow**, run from the repo root with each tree as a stow package:

```sh
stow --no-folding common
stow --no-folding linux   # or macos
```

**Always pass `--no-folding`** so stow symlinks individual files instead of folding whole directories into a single symlink. This keeps `$HOME` config dirs as real directories that can mix stowed and non-stowed files (and lets new files be stowed without breaking existing links). `.stow-local-ignore` controls what stow skips — notably most of `common/.config/fish` and `common/.config/nushell` are ignored except an explicit whitelist of hand-maintained files.

`link.sh` is a legacy alternative deployer (per-file `ln -svfr` symlinks) and is not the current method — prefer stow.

## Bootstrapping a machine

`install/install.sh` provisions a fresh Arch system end to end:

1. pacman base-devel/git/rustup → rustup stable → builds `paru` (AUR helper) from source.
2. Installs packages via `install_packages` from `install/packages/*.packages`, in order: `base.packages`, `gui.packages`, `aur.packages`. Each file is plain package names; `#` comments and blank lines are stripped. Installs error out hard if any package fails to register.
3. Installs `mise`, the SDDM Catppuccin theme (`sddm.sh`), and the Bluetooth USB autosuspend fix (`btusb.sh`, symlinks a modprobe conf).
4. Clones the separate neovim config from `github.com/lorenzorapetti/nvim` into `~/.config/nvim` (nvim is **not** vendored here).
5. Symlinks configs (the script still sources `link.sh`; if deploying by hand use `stow --no-folding` per the section above), enables systemd **user** services (wayle, vicinae, udiskie, ssh-add, hypridle, hyprmoncfgd), and opens KDE Connect ports in ufw.

When adding a program, put its package in the matching `packages/*.packages` file *and* its config under the right tree — both are required for a clean install to reproduce it.

## Hyprland config is Lua, not hyprlang

`linux/.config/hypr/hyprland.lua` is the entrypoint. It `require`s the `hyprland.*` modules (`autostart`, `environment`, `keybinds`, `rules`, `input`, `style`) and calls into the global `hl` API (e.g. `hl.config{...}`). This relies on Hyprland's Lua config support; stubs live at `/usr/share/hypr/stubs` (see `.luarc.json`). `monitors.lua` is generated by `hyprmoncfg` and is gitignored — don't edit or commit it. Format Lua with the repo's `stylua.toml`.

## Shell

fish is the primary shell (`common/.config/fish/`), with nushell also configured. `config.fish` sources `aliases.fish` and `utils.fish` and initializes mise, fzf, zoxide, direnv, atuin, and television. Most of `common/.config/fish/` is gitignored except the hand-maintained files whitelisted in `.stow-local-ignore`.
